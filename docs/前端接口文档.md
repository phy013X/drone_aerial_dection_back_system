# 前端接口文档

## 1. 认证模块

### 1.1 用户登录
- **接口地址**：`/auth/login`
- **请求方法**：`POST`
- **请求参数**：
  | 参数名 | 类型 | 必填 | 描述 |
  |-------|------|------|------|
  | username | string | 是 | 用户名 |
  | password | string | 是 | 密码 |
- **成功返回**：
  ```json
  {
    "code": 200,
    "data": {
      "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
      "user": {
        "id": 1,
        "username": "admin",
        "name": "管理员",
        "role": "admin",
        "avatar": "https://example.com/avatar.jpg"
      }
    },
    "message": "登录成功"
  }
  ```
- **失败返回**：
  ```json
  {
    "code": 401,
    "message": "用户名或密码错误"
  }
  ```

### 1.2 用户注册
- **接口地址**：`/auth/register`
- **请求方法**：`POST`
- **请求参数**：
  | 参数名 | 类型 | 必填 | 描述 |
  |-------|------|------|------|
  | username | string | 是 | 用户名 |
  | password | string | 是 | 密码 |
  | name | string | 是 | 姓名 |
  | email | string | 否 | 邮箱 |
  | phone | string | 否 | 手机号 |
- **成功返回**：
  ```json
  {
    "code": 200,
    "data": {
      "id": 2,
      "username": "newuser",
      "name": "新用户"
    },
    "message": "注册成功"
  }
  ```
- **失败返回**：
  ```json
  {
    "code": 400,
    "message": "用户名已存在"
  }
  ```

### 1.3 获取用户信息
- **接口地址**：`/auth/user`
- **请求方法**：`GET`
- **请求参数**：无
- **成功返回**：
  ```json
  {
    "code": 200,
    "data": {
      "id": 1,
      "username": "admin",
      "name": "管理员",
      "role": "admin",
      "avatar": "https://example.com/avatar.jpg",
      "email": "admin@example.com",
      "phone": "13800138000"
    },
    "message": "获取成功"
  }
  ```
- **失败返回**：
  ```json
  {
    "code": 401,
    "message": "未授权"
  }
  ```

### 1.4 更新用户信息
- **接口地址**：`/auth/user`
- **请求方法**：`PUT`
- **请求参数**：
  | 参数名 | 类型 | 必填 | 描述 |
  |-------|------|------|------|
  | name | string | 否 | 姓名 |
  | email | string | 否 | 邮箱 |
  | phone | string | 否 | 手机号 |
  | avatar | string | 否 | 头像URL |


## 12. 通用响应格式

所有接口的响应格式统一如下：

### 成功响应
```json
{
  "code": 200,
  "data": {}, // 具体返回数据
  "message": "操作成功"
}
```

### 失败响应
```json
{
  "code": 错误码,
  "message": "错误信息"
}

错误码说明：
- 400: 参数错误
- 401: 未授权
- 403: 禁止访问
- 404: 资源不存在
- 500: 服务器内部错误
```

## 13. 认证方式

所有需要认证的接口都需要在请求头中携带 token：

```
Authorization: Bearer {token}
```

其中 `{token}` 是登录成功后返回的 JWT token。
