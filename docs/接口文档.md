# 无人机航拍实时检测系统接口文档

## 1. 接口概述

本文档描述了无人机航拍实时检测系统的API接口，包括前端与后端之间的通信接口。系统采用RESTful API设计风格，使用JSON格式进行数据交换。

### 1.1 基础URL

```
http://localhost:8080/api
```

### 1.2 认证方式

系统采用JWT（JSON Web Token）进行认证，认证流程如下：

1. 用户登录获取token
2. 在后续请求的请求头中携带token：`Authorization: Bearer {token}`

## 2. 接口列表

### 2.1 用户管理

| 接口名称 | 请求方法 | URL | 功能描述 |
|---------|---------|-----|--------|
| 用户登录 | POST | /auth/login | 用户登录获取token |
| 用户注册 | POST | /auth/register | 注册新用户 |
| 获取用户信息 | GET | /auth/user | 获取当前用户信息 |
| 更新用户信息 | PUT | /auth/user | 更新用户信息 |

### 2.2 设备管理

| 接口名称 | 请求方法 | URL | 功能描述 |
|---------|---------|-----|--------|
| 获取设备列表 | GET | /devices | 获取所有设备列表 |
| 获取设备详情 | GET | /devices/{id} | 获取设备详细信息 |
| 添加设备 | POST | /devices | 添加新设备 |
| 更新设备 | PUT | /devices/{id} | 更新设备信息 |
| 删除设备 | DELETE | /devices/{id} | 删除设备 |
| 设备状态更新 | POST | /devices/{id}/status | 更新设备状态 |

### 2.3 检测管理

| 接口名称 | 请求方法 | URL | 功能描述 |
|---------|---------|-----|--------|
| 实时检测 | POST | /detection/realtime | 实时检测图像 |
| 获取检测历史 | GET | /detection/history | 获取检测历史记录 |
| 获取检测详情 | GET | /detection/{id} | 获取检测详细信息 |
| 分析检测结果 | POST | /detection/analyze | 分析检测结果 |

### 2.4 告警管理

| 接口名称 | 请求方法 | URL | 功能描述 |
|---------|---------|-----|--------|
| 获取告警列表 | GET | /alerts | 获取告警列表 |
| 获取告警详情 | GET | /alerts/{id} | 获取告警详细信息 |
| 处理告警 | PUT | /alerts/{id}/process | 处理告警 |
| 关闭告警 | PUT | /alerts/{id}/close | 关闭告警 |

### 2.5 系统管理

| 接口名称 | 请求方法 | URL | 功能描述 |
|---------|---------|-----|--------|
| 获取系统状态 | GET | /system/status | 获取系统运行状态 |
| 获取系统配置 | GET | /system/config | 获取系统配置 |
| 更新系统配置 | PUT | /system/config | 更新系统配置 |
| 获取系统日志 | GET | /system/logs | 获取系统日志 |

## 3. 详细接口描述

### 3.1 用户管理接口

#### 3.1.1 用户登录

**请求方法**：POST
**URL**：/auth/login
**请求参数**：

| 参数名 | 类型 | 必选 | 描述 |
|-------|------|------|------|
| username | string | 是 | 用户名 |
| password | string | 是 | 密码 |

**请求示例**：

```json
{
  "username": "admin",
  "password": "123456"
}
```

**响应示例**：

```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
      "id": 1,
      "username": "admin",
      "role": "admin",
      "name": "管理员"
    }
  }
}
```

#### 3.1.2 用户注册

**请求方法**：POST
**URL**：/auth/register
**请求参数**：

| 参数名 | 类型 | 必选 | 描述 |
|-------|------|------|------|
| username | string | 是 | 用户名 |
| password | string | 是 | 密码 |
| name | string | 是 | 姓名 |
| role | string | 是 | 角色 |

**请求示例**：

```json
{
  "username": "user1",
  "password": "123456",
  "name": "用户1",
  "role": "user"
}
```

**响应示例**：

```json
{

    "id": 2,
```

## 4. 错误处理

### 4.1 错误响应格式

```json
{
  "code": 错误码,
  "message": "错误信息",
  "data": null
}
```

### 4.2 常见错误码

| 错误码 | 描述 |
|-------|------|
| 400 | 请求参数错误 |
| 401 | 未授权，需要登录 |
| 403 | 禁止访问，权限不足 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |
| 503 | 服务不可用 |

### 6.2 接口版本控制

系统采用URL路径版本控制，例如：

```
http://localhost:8080/api/v1/auth/login
http://localhost:8080/api/v2/auth/login
```

### 6.3 速率限制

系统对API调用进行速率限制，防止滥用：

- 普通用户：60次/分钟
- 管理员：120次/分钟
- 设备接口：300次/分钟

### 6.4 跨域处理

系统支持CORS（跨域资源共享），允许前端应用从不同域名访问API：

```
Access-Control-Allow-Origin: *
Access-Control-Allow-Methods: GET, POST, PUT, DELETE, OPTIONS
Access-Control-Allow-Headers: Content-Type, Authorization
```
